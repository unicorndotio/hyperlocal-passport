# ADR-004: Coupon-Based Validation System

## Status

Accepted

## Date

2026-05-17

## Context

Inicialmente, a validação no caixa havia sido desenhada com um QR Code dinâmico
baseado em JWT focado em provar a identidade do usuário (um "crachá digital").
No entanto, identificamos a necessidade de oferecer benefícios muito mais
granulares para lojistas e usuários: descontos limitados mensalmente por
usuário, ofertas especiais globais (ex: "10 cupons de 15% off"), ou benefícios
de data específica (ex: "Chopp em dobro na sexta-feira").

## Decision

Decidimos alterar a arquitetura de validação para um **Sistema de Cupons (Coupon
Codes)**.

- Em vez de um "QR Code de Usuário", teremos um "QR Code de Cupom".
- Lojistas criarão `Coupons` (ofertas básicas com limites mensais ou ofertas
  especiais com limites globais e de data).
- O usuário deve "resgatar" (redeem) a oferta no sistema, gerando uma
  `Redemption` com um código alfanumérico único (ex: `JUR-8B9X`).
- O aplicativo do morador gera o QR Code com esse código único.
- O lojista escaneia o QR Code ou simplesmente digita o código alfanumérico no
  painel para validar a transação (útil para quem não tem leitor ou câmera no
  caixa).

## Alternatives Considered

### Alternative 1: QR Code Dinâmico (Time-based JWT)

- **Description**: O QR code apenas prova que o usuário está lá, e o sistema no
  caixa aplica a oferta padrão da loja.
- **Pros**: O morador só precisa abrir o app e mostrar a tela.
- **Cons**: Não permite concorrência por escassez (ex: "os primeiros 10 ganham
  desconto maior"), não evita acúmulo de benefícios (stacking), e é rígido
  demais para campanhas promocionais atrativas.
- **Why rejected**: O sistema de cupons traz muito mais valor comercial para o
  lojista criar campanhas atrativas de engajamento na comunidade do que um
  desconto fixo eterno.

## Consequences

### Positive

- Flexibilidade total de marketing para os lojistas.
- Usuário não consegue "empilhar" descontos (1 cupom = 1 transação).
- Facilita a operação no caixa: o lojista pode apenas digitar um código curto em
  vez de obrigatoriamente precisar de um leitor de QR Code.

### Negative

- Adiciona uma etapa extra para o morador: em vez de apenas mostrar a tela
  inicial, ele precisa ativamente clicar em "Resgatar Desconto" na página da
  loja antes de ir ao caixa.
- Adiciona complexidade no banco de dados (controle de concorrência no Deno KV
  para cupons com limite global).

## Implementation Notes

- O código do cupom gerado (`Redemption ID`) deve ser curto e de fácil digitação
  (ex: 6 a 8 caracteres amigáveis, removendo letras confusas como O e 0).
- Precisaremos utilizar transações atômicas no Deno KV (`kv.atomic()`) ao
  resgatar cupons limitados globais para evitar _race conditions_ (duas pessoas
  pegarem o último cupom ao mesmo tempo).
