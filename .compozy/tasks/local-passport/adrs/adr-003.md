# ADR-003: Document Upload Storage

## Status

Accepted

## Date

2026-05-17

## Context

O processo de aprovação de moradores exige o upload de dois documentos comprobatórios (documento com foto e comprovante de residência). O Deno KV possui um limite estrito de 64KB por chave-valor, o que impossibilita o armazenamento direto de imagens.

## Decision

Decidimos utilizar o **DigitalOcean Spaces** (Object Storage S3-compatible) para o armazenamento de todos os arquivos binários estáticos submetidos pelos usuários e imagens de logotipos das empresas parceiras.

## Alternatives Considered

### Alternative 1: Local Filesystem Storage (`/static/uploads`)
- **Description**: Salvar as imagens diretamente na pasta local do servidor Deno.
- **Pros**: Nenhum custo adicional ou SDK necessário.
- **Cons**: Impede o deploy em ambientes serverless efêmeros (como Deno Deploy) onde o sistema de arquivos é read-only.
- **Why rejected**: Limitaria fortemente a escalabilidade e opções de hospedagem.

### Alternative 2: Armazenamento Base64 em KV
- **Description**: Comprimir fortemente as imagens e salvá-las codificadas em Base64 divididas em múltiplas chaves no Deno KV.
- **Pros**: Mantém tudo dentro do ecossistema Deno.
- **Cons**: Muito complexo, oneroso computacionalmente e custoso em termos de leitura/gravação no banco de dados.
- **Why rejected**: Má prática arquitetural.

## Consequences

### Positive
- Permite o uso de CDNs na frente do Storage para carregamento rápido dos logotipos.
- Escalonamento infinito para documentos sem inflar o banco de dados principal.
- Integração padrão usando SDKs AWS/S3.

### Negative
- Necessidade de gerenciar credenciais de API (Access/Secret keys) adicionais.
- A aplicação precisará implementar geração de URLs pré-assinadas (pre-signed URLs) para visualização segura de documentos sensíveis pela equipe de aprovação.

## Implementation Notes
- Usaremos as credenciais padrão do AWS S3 SDK (ou um cliente HTTP customizado via Deno fetch compatível com S3) conectando ao endpoint do DigitalOcean.
